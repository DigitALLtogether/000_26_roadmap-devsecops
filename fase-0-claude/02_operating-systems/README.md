## Objectivo

Construir um AI Operating System pessoal sobre Claude Code — a camada de inteligência que substitui o saltar entre aplicações e centraliza contexto, ligações, capacidades e cadência numa só interface.

## Frameworks principais

### 3 M's
| M | Conceito |
|---|---------|
| **Mindset** | Default shift, function breakdown, curiosity rule |
| **Method** | Como decidir o que automatizar e em que percentagem |
| **Machine** | A arquitectura técnica do AI OS |

### 4 C's do AI OS
| C | O que é | Exemplo prático |
|---|---------|----------------|
| **Context** | O que o AI sabe sobre ti, os teus objectivos, ferramentas, voz | CLAUDE.md, perfil pessoal, notas de estudo |
| **Connections** | A que dados consegue chegar (MCPs, APIs, CLIs) | Gmail, Google Drive, GitHub |
| **Capabilities** | O que consegue produzir com esses dados | Gerar resumos, criar labs, actualizar READMEs |
| **Cadence** | Quando age autonomamente enquanto trabalhas ou dormes | Automações agendadas |

> Os 4 C's constroem-se por ordem: Context → Connections → Capabilities → Cadence.

## Mindset de automação

- **Default shift:** antes de qualquer tarefa perguntar "como pode AI fazer pelo menos 30% disto?"
- **Function breakdown:** decompor tarefas grandes em blocos pequenos e automatizáveis
- **Curiosity rule:** tratar AI como mentor — perguntar sempre *porquê* foi feito assim
